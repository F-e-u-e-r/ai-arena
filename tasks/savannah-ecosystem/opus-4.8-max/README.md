# 非洲大草原生態系統 — Canvas 2D（Opus 4.8 · max）

原生 HTML + CSS + JavaScript、純 Canvas 2D 的非洲大草原生態動畫：獅群、象群與獨行的豹依真實習性
在多層視差草原上循環移動，搭配 6 種天氣 × 乾濕兩季的生態系統；閃電永不擊中動物、龍捲風追逐最近的
獵物、粒子以物件池管理。單一自包含 `index.html`，無任何框架、遊戲引擎或外部函式庫。

---

## 如何啟動

直接用瀏覽器開啟 `index.html` 即可（無外部相依），或用簡單 static server：

```bash
python3 -m http.server 8000
# 開啟 http://localhost:8000/tasks/savannah-ecosystem/opus-4.8-max/index.html
```

### 操作

| 動作 | 說明 |
| --- | --- |
| ☀️🌦️🌧️❄️⛈️🌪️ / 數字 `1`–`6` | 切換天氣（單選）：晴天／下雨／大雨／下雪／雷暴／龍捲風 |
| 🌿 / `S` | 切換乾／濕季（每 60 秒亦會自動輪轉） |
| 💭 / `M` | 開關動物頭頂的狀態（心情）標籤 |
| 📊 / `D` | 開關 FPS／粒子數／動物數／活躍度除錯資訊 |

`❄️ 下雪` 只在**濕季**可用（乾季按鈕停用），對應題目「雪只在乾季較少出現」。

## 為何選擇 Canvas 2D

題目指定必須用 Canvas 2D。相較 DOM/SVG，Canvas 2D 以單一 immediate-mode 繪圖表面，
每幀重畫上百個粒子與多隻帶走路循環的動物時，沒有大量 DOM node 的重排／重繪成本，
最適合本題「多粒子系統 + 大量重繪 + 長時間運行」的需求。所有動物與粒子皆以基本圖形
（橢圓、線段、貝茲曲線、路徑）即時繪製，無需任何 sprite 素材。

## 程式架構（模組化 OOP）

| 類別 | 職責 |
| --- | --- |
| `ObjectPool` | 通用物件池：`acquire()`／`release()`，粒子重複利用不觸發 GC |
| `ParticleSystem` | 以池為基礎的粒子系統（雨／大雨／雪／水花／塵／碎屑／水氣） |
| `SeasonManager` | 乾／濕季，60 秒自動輪轉，控制配色、亮度、天氣可用性、活躍度、群聚度 |
| `WeatherManager` | 6 種天氣狀態機、平滑過渡、閃電安全區、龍捲風生成 |
| `Tornado` | 追逐最近動物 + 捲起地面粒子 |
| `Animal` / `Lion` / `Elephant` / `Leopard` | 角色系統：狀態機 + 走路循環 + 陰影 + 心情標籤 |
| `Herd` | 群體行為（獅群／象群共同前進、一起休息、季節性聚散） |
| `Environment` | 背景視差、草原、水塘、金合歡樹、雲、前景草叢 |
| `Game` | 場景組裝、`requestAnimationFrame` + `deltaTime` 主迴圈、UI/RWD |

主迴圈使用 `requestAnimationFrame` 搭配以 `performance.now()` 算出的 `deltaTime`，並把
`dt` 夾在 0.05 秒內，避免分頁切回時的大跳幀造成物理爆走。所有動作、粒子、過渡皆以 `dt`
計算，因此速度與 FPS 無關。

## 動物習性實作

- **獅子（群體、慢速、偶爾休息／巡邏）**：3 隻組成獅群（1 公有鬃毛、2 母）。由 `Herd` 統一
  控制「行走／休息」狀態——偶爾整群趴下休息（`state='rest'`，四肢收起、身體壓低），其餘時間
  緩慢前進。乾季更常休息、聚得更緊。
- **大象（龐大、緩慢、成群、喝水／抬鼻）**：3 隻象群，體型最大、速度最慢。經過水塘且在近景時
  有機率進入 `drink` 狀態——低頭、鼻子捲下到水面並濺起水花；也會偶爾 `trunk` 抬鼻。耳朵持續拍動。
- **豹（單獨、快速、隱密、爬樹）**：2 隻各自獨立行動，速度最快，移動時略降不透明度表現隱密感。
  偶爾走向最近的金合歡樹並**垂直攀爬**（`toTree → climbUp → perch → climbDown`），在樹上停留
  後再下來。爬樹時身體貼著樹幹近垂直定位。
- **循環移動**：所有動物由左側進入，抵達右側後由左側無縫重現；重現時 depth（遠近）微幅變化，
  形成不同的體型與速度層次。
- **`depth`（0 遠 → 1 近）** 決定每隻動物的 y 座標、縮放與速度，並依 y 排序繪製，讓前後遮擋正確。

## 季節變化邏輯

`SeasonManager` 以 `blend ∈ [0,1]`（乾 0 → 濕 1）在 2.5 秒內平滑過渡，對外提供內插後的配色與衍生值：

- **視覺**：天空上下色、遠／近草地色、草葉色、太陽色、整體亮度都隨 `blend` 內插——乾季偏黃褐、
  朦朧、炎熱；濕季偏翠綠、清亮。
- **天氣可用性**：`allows('snow')` 僅在濕季回傳 true（乾季雪按鈕停用）。
- **動物行為**：`activity`（濕季 0.95 / 乾季 0.5）拉高濕季的移動速度與動作頻率；`cohesion`
  在乾季較高，使獅群／象群的 depth 往群體平均靠攏、聚得更緊（旱季群聚習性）。
- **自動輪轉**：每 60 秒自動切換乾濕季，狀態列上的進度條顯示本季剩餘時間；也可手動切換。

## 天氣系統與粒子管理效能策略

- **物件池（Object Pooling）**：`ParticleSystem` 由 `ObjectPool` 預先配置 400 個粒子空殼，
  `emit()` 取用、壽命到或落地時 `release()` 歸還。實測大雨下同時 ~180 個活躍粒子，池大小恆定
  在 400、JS heap 穩定，**完全不因粒子生滅產生 GC 卡頓**。
- **天氣切換清理**：`WeatherManager.set()` 會呼叫 `particles.clear()` 把所有活躍粒子一次歸還
  池中（實測切換後活躍數歸零），並重設積雪、視需要建立／移除龍捲風——舊粒子被正確回收而非外洩。
- **平滑過渡**：天氣強度 `intensity` 與天空壓暗 `darkness` 都以時間常數趨近目標值，切換天氣時
  降水密度與整體氛圍是漸變而非瞬變。
- **次要效果**：雨滴落地濺起水花並累積地面積水（`wetness` → 反光水漥）；下雪累積地面白雪
  （`snowAccum`）；龍捲風從地面捲起塵土與碎屑。
- **固定發射步進**：粒子以固定 1/60 秒步進發射，避免高刷新率螢幕狂噴粒子。

## 雷暴安全區演算法（核心）

目標：**閃電永遠不會擊中任何動物**。每次擊發前動態計算安全落點：

1. 蒐集所有動物的即時 x 與**危險半徑**（體寬一半 + 46px 安全邊界）。
2. 在畫面寬度上取 18 個等距候選點 + 6 個隨機候選點。
3. 對每個候選點計算「到所有動物危險區的最小間隙」
   `gap = min over animals(|candidateX − animalX| − dangerRadius)`。
4. 取 `gap` 最大的候選點作為落點；**只有當 `gap > 0`（確定落在所有動物危險區之外）才真的擊發**，
   否則本次跳過——從構造上保證絕不打到動物。

實測：讓動物隨機散佈全畫面、連續 200 次擊發，169 次擊發／31 次因無安全間隙而跳過，
**所有擊發落點的最小間隙皆 > 0，沒有任何一次落入動物危險區**。開啟除錯資訊時，雷暴下會以紅線
畫出每隻動物的危險區、以綠線標出所選的安全落點，可直接觀察演算法決策。

## 龍捲風追逐

`Tornado` 每幀找出最近的動物作為目標，朝它移動；移動速度（~70 px/s）刻意低於動物的逃跑速度，
形成「追逐但追不上」。動物在龍捲風接近至 190px 內時進入逃跑（加速並遠離龍捲風、頭頂顯示 😰）。
粒子系統另外對漩渦附近的塵土／碎屑／雨滴施加「向中心吸 + 往上捲 + 切向旋轉」的力，呈現捲起效果。

## Responsive Embed

Canvas 以 DPR 縮放並在 `resize` 時重算版面與元素座標；`uiScale` 依畫面大小調整動物／樹／
粒子尺寸。控制列為單排 icon 按鈕，`@media (max-width: 520px)` 時縮小尺寸。在 320×200、16:10 的
預覽下，主場景仍完整可見，6 個天氣 + 季節／狀態／除錯共 9 顆控制鈕單排置中、不重疊、不溢出，
狀態列精簡於左上、不遮住主要內容；動物過小時自動隱藏心情標籤以免雜亂。
