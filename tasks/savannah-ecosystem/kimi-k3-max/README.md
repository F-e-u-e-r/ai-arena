# 非洲大草原生態系統 (Canvas 2D) — Kimi K3 max

以原生 HTML + JavaScript + CSS 打造的 2D 非洲大草原生態系統動畫:8 隻野生動物(獅群 ×3、象群 ×3、豹 ×2)依真實習性活動,6 種天氣 × 乾/濕兩季即時切換,閃電永遠避開動物、龍捲風會追逐最近的動物。

## 如何啟動

- 直接用瀏覽器開啟 `index.html` 即可(所有腳本皆為 classic script,`file://` 下可正常運作);
- 或使用任意 static server,例如 `python3 -m http.server` 後開啟 `http://localhost:8000/`。
- 無任何外部相依(無框架、無 CDN、無圖片資源),離線可跑。

## 為何選擇 Canvas

本題需要同時管理 **數百至上千個粒子**(暴雨 620 滴/秒、水花、雪花、龍捲風碎屑、沙塵)加上 8 隻逐幀程序化繪製的動物與多層視差背景:

- Canvas 2D 是**立即模式 (immediate mode)** 渲染:每幀的繪製內容完全由程式決定,粒子生滅只需更新資料,不需要像 DOM/SVG 那樣同步維護大量節點,天生適合高頻生滅的粒子場景;
- 可以對同類粒子做**批次繪製**(全部雨滴收成單一 `Path2D` 概念的一次 `stroke()`),把 draw call 從 O(n) 降到 O(1),這在 DOM/SVG 上做不到;
- 調色盤插值、動物步態、天氣過渡全都是每幀數值運算,Canvas 讓「資料 → 像素」的路徑最短,長時間運行也好控制記憶體。

## 系統架構

```
index.html            畫布 + HUD + 控制列,依序載入 js/
style.css             版面與 320×200 嵌入的自適應樣式
js/utils.js           數學/色彩工具(clamp、lerp、混色、指數平滑)
js/particles.js       ParticlePool(Object Pooling)+ PrecipSystem(雨/雪/水花/碎屑/塵)
js/environment.js     調色盤(季節×天氣混合)、Background(視差背景)、LightningSystem、Tornado
js/animals.js         Animal 基類 + Lion/Elephant/Leopard + AnimalGroup(群體狀態機)
js/weather.js         WeatherManager(天氣數值平滑過渡)、SeasonManager(60s 乾濕季輪轉)
js/main.js            Canvas/DPR 設定、rAF + deltaTime 主迴圈、UI 與鍵盤、渲染分層
```

所有尺寸皆以 `u = 畫高 / 400` 為單位縮放,動物另有 lane(深度道)決定縮放與地面 y,因此在 320×200 嵌入預覽與全螢幕下構圖一致。

## 粒子管理與效能策略

1. **Object Pooling(必考實作)**:`ParticlePool` 預先配置固定容量的物件(雨 900 / 水花 320 / 雪 380 / 碎屑 240 / 塵 140)。`spawn()` 從 `freeList` pop 一個槽位;粒子死亡時以 **swap-remove** O(1) 還回。整個生命週期 **0 配置**,長時間運行沒有 GC 壓力;池滿時直接丟棄新粒子,保幀率優先。
2. **批次繪製**:全部雨滴收成單一 path 一次 `stroke()`;雪花單一 `fill()`;水花同理。每幀 draw call 數與粒子數無關。
3. **累加器生成**:以 `rate × dt` 累加決定每幀生成數,速率由 `WeatherManager` 平滑插值——天氣切換時生成率連續變化,舊型粒子停止生成後**自然死亡並回池**,不需要額外清理動作。
4. **無每幀配置**:主迴圈內不產生新陣列/閉包(動物排序僅 8 元素);天空/地面漸層做內容快取,顏色沒變不重建 gradient;調色盤全程以 `[r,g,b]` 陣列運算,每幀只轉一次字串。
5. **deltaTime 有上限**(`≤ 50ms`):切分頁回來不會暴衝;DPR 上限 2,兼顧清晰度與 fill rate。

## 雷暴安全區演算法

目標:**閃電永遠不會擊中任何動物**,且落點自然分散。

1. 把畫面寬度切成 28 個候選落雷點(避開邊緣 5%);
2. 對每個候選點,計算它與所有動物的**淨水平距離** `|x − animal.x| − animal.halfWpx`(動物半寬依其 lane 縮放後的實際像素),取最小值作為該點的「危險度」——越遠越安全;
3. 以「真實距離 + 小抖動」排序挑選落點(抖動只影響挑選、避免每次落在同一條垂直線上);
4. 最後用**真實距離**(不含抖動)驗證:若最佳點仍小於最小安全距離 `SAFE = max(46px, u×80)`,本輪**不落雷**,改為雲內閃光(只有螢幕閃光、沒有落地雷)。因此無論動物怎麼移動,落地雷與最近動物的距離恆 ≥ SAFE——本演算法另以 3000 次隨機布局的自動測試驗證過此性質。

雷電本體用**中點位移法**生成 4 次迭代的曲折折線,附 1–2 條分支,外層暈光 + 內層亮芯兩次描邊;落雷瞬間觸發 `onFlash()`,全場動物受驚短暫加速。

## 動物習性實作與季節變化邏輯

| 物種 | 習性實作 |
| --- | --- |
| **獅子**(3 隻) | 群體移動:`AnimalGroup` 領頭公獅(有鬃毛)決定速度與狀態,成員以 offset 跟隨;偶爾進入 `REST`(依序躺下 💤),乾季休息機率更高(保存體力)、濕季更常巡邏;風暴/龍捲風時不休息 |
| **大象**(3 隻) | 成群緩慢移動(最慢),經過水窪時成群 `DRINK`(鼻子平滑伸入水中 💧,乾季更渴、機率更高);行走中偶爾抬鼻噴水 💦;耳朵持續搧動 |
| **豹**(2 隻) | 獨行、速度快且低伏(身體貼地的潛行步態);靠近金合歡樹時進入 `CLIMB_UP → SIT → CLIMB_DOWN` 垂直攀爬動畫 🌳,下樹後冷卻 10–22 秒;雪天與風暴不爬樹 |
| 全部 | 由左側進入、右側離場後無縫回到左側;龍捲風進入驚嚇半徑時朝反方向逃逸 💨(可短暫向左,不停止、不重置位置);閃電時受驚加速 ❗ |

**天氣對行為的影響**(每種天氣都有 `speedMul` 與 `activity` 兩個行為因子):雪天全体减速 30%、雨天動物加快脚步、雷暴與龍捲風時神经质加速且無法休息/喝水/爬樹。

**季節對行為的影響**(`SeasonManager.blend` 在乾 0 ↔ 濕 1 之間以約 2.5 秒平滑過渡):

- `activity` 0.8 → 1.3:濕季動物更活躍(特殊動作頻率提高);
- `cluster` 0.55 → 1.05:乾季群聚更緊(群體 offset 縮小),濕季分散;
- `speedMul` 0.92 → 1.12:濕季略快;
- 視覺同步:草地金黃 ↔ 翠綠、水窪變大、濕季開花、整體亮度;季節每 60 秒自動輪轉(按鈕上有倒數),也可手動切換,**切換時動物不停止、不重置位置**;
- 天氣可用性:雪只在乾季出現(濕季按鈕停用;若下雪中進入濕季,自動平滑退回晴天)。

## 天氣切換的平滑過渡

`WeatherManager` 內部維護一組「目前數值」(`cur`:雲量、暗度、風速、各粒子生成率、行為因子…),每幀以指數平滑逼近目標天氣的定義值;調色覆蓋層另有「舊天氣淡出 + 新天氣淡入」的交叉淡化(`prev.effect` 衰減、`cur.effect` 上升),因此任何切換——含 6 種天氣互切與季節輪轉——畫面都是連續漸變,不會瞬間跳色。次要效果:雨天地面 `wetness` 累積出積水窪(放晴後消退)、雪天 `snowCover` 讓大地與山頂積雪、龍捲風捲起螺旋碎屑與沙塵。

## Object Pooling 實作細節

```text
slots[capacity]  固定預配置物件(粒子欄位重用)
freeList         閒置槽位 stack;live[] 密集存放存活粒子
spawn()          freeList.pop() → 標記 active → push 進 live;池滿回傳 null(丟棄)
releaseAt(i)     標記 inactive → 槽位還回 freeList → live 以 swap-remove O(1) 移除
不變式           freeList.length + live.length === capacity(自動化測試持續斷言)
```

更新迴圈一律**反向走訪** `live[]`,死亡即 `releaseAt(i)`,沒有陣列搬移、沒有配置、沒有記憶體洩漏;長跑 35 分鐘虛擬時間的自動測試中,池守恆與動物座標皆穩定。

## 操作方式

- 天氣按鈕(單選):☀️ Sunny / 🌧️ Rain / 🌧️ Heavy Rain / ❄️ Snow / ⛈️ Thunderstorm / 🌪️ Tornado,快捷鍵 `1`–`6`;
- 季節按鈕:🌾 乾季 ↔ 🌿 濕季(含 60 秒自動輪轉倒數),快捷鍵 `S`;
- `D`:FPS / 粒子數 / 動物數 debug overlay;
- 320×200 嵌入預覽下按鈕自動收成圖示單行,不遮主要場景。
